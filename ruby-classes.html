Understanding Ruby Class

What is a Ruby class?
A ruby class is the associated “type” linked to individual objects. The class gives the objects is is related to certain traits and properties that run through all objects associated with that class.
Let’s use a real world example.
Let’s say a Ford Brand Card is a specific object. It is similar to other cars and is a car (class) itself. You can say all cars are a type of vehicle (super class).
So, we know certain attributes are associated with being a vehicle. Cars and trucks both carry passengers. Cars and airplanes both hold cargo. So, some shared traits across the board when you associate an object being a vehicle.
Now, we know there’s a difference between cars and trucks. Cars are smaller than trucks and trucks are usually used to haul larger cargo. In turn, we know specific details are shared with all cars. Cars do not have a bed or cargo box like trucks. Cars are usually around the same size as other cars.
Finally, we understand certain aspect about the type of object a Ford Brand Car is. It has the Ford brand warranties. It has the ford logo on it somewhere and associates with the ford assigned model type you can use to reference it.
So,similarly, ruby objects inherit certain selected traits / characteristics from its class and its superclass and so on.

Examination of ruby class / method codes

Below is a simple class code that will process a method upon initializing.
class Greeting => #We define the class as Greeting. def hello => #We define a method for the class as hello. puts "Hello guy!" => #The code the method will run. end end

So, above we created a class called Greeting and defined a method for that class to run a code.
To run the above code we use:
test = Greeting.new test.hello

In order to run the code, we must first initialize the Greeting class for a variable we want to use. Then, we directly call the method using that variable.
Additionally we can use different types of variables throughout our class.
Talking about type of varialbes!
  1. Local Variables (variable) - Are the most common variables but can only be used in the method they are defined in.
  2. Instance Vairables (@variable) - That can be used across methods for a particular instance or object
  3. Class Variables (@@variable) - That are available across objects of a whole class it is asociated with.
  4. Global Variables ($variable) - That are available across classes.
$one = 1 class Math @@two = 2 def print @three = 3 p $one p @@two p @three end def print2 p @three end end test = Math.new test.print test.print2

If you examine the code above, we can still print out $one even though it is outside the class because it is a global variable.
We can print out @@ two even though it is outside the method because it is a class variable.
If you review print2 method, we did not define any variables there, but; because we defined the instance variable @three within the first print method, we can use it in print2.
