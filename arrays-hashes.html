Array Vs Hash. When to use each?



To start answering this question, we must first understand the finer details of an array and hash in ruby.

An array is a list of data; a collection.

We can use an array empty array to store our strings, integers, and variables. This is useful for creating stores of data we need to access in a concise location (in an array).

To start an array, we use bracket or “[ ]”.

We can also call on an individual element in an array by calling on the element’s “index” or location in the array.

EX:
array[0] will provide the very first element of an array. Likewise: array[1] will call the 2nd element of an array 

Array = [“cat”, “dog”, “meow”]

array[2] = “meow”

A hash is a collection of pairs of information. And each pair is made up of a “key” and “value”.

Or, each “key” (name) has a “value” (data).

We use the key to get the value paired to it.
EX:

toys_hash = { “ball” => “red”, “doll” => “girl”, “ valve => “best”}

So if we call, toys_hash[“ball”], we will receive the key “red”.

So why do we use a hash instead on an array?

Hashes can can have any object as a key. Whereas, arrays use an integer (index) as a key.

So, while it’s easier to access array elements, hashes provide more flexibility in its naming / storage of data.

Thus, Array and Hashes are both useful for storing data, but; you must consider ease of access (array) vs flexibility (hash) when considering your codes needs!
