The Importance of Ruby Map!


This blog post is designed to review and understand one of ruby’s most important enumerable: map.

What is an enumerable ruby method?


To answer what map is, we need to first understand what an enumerable is.

An enumerable is actually a pack of methods that ruby programmers thought were useful and packages them together with other objects. An enumerable was created so programmers do not have to continuously write a pack of methods they need to commonly use.

So what does the “.map” Enumerable do for us?


The “,map” enumerable returns a new array that contains the output / results of running the block of code once over a given set of elements.

Example
(1..4).map { |x| x * x } #=> [1,4,9,16]


In the code above, “.map” returns an array containing the results of multiplying the range of integers inclusively between 1..4.

Thus we get 1 * 1 = 1 as the first element in index(0). We get 2 * 2 = 4 as the 2nd elment in index (1). And, etc.

So, we use .map to create a new array based on the original array with the values modified by the block of code we gave it. In this case, the code was multiplying each element by itself.

So, “.map” is an effective way of easily running a desired code block over a specified array or hash and get a new array in return.

This is useful if you want a non-destructive return of code!
