Ruby Enumerables

Ruby allows for "mixins" of code. I.E. when we write a code and want to share it with a bunch of unrelated classes, the code can be put ino a module and "mixed in" with the other classes. This ability of setting methods within modules allows for ruby to run Enumerables.

In order for a class to use an Enumerable, the class must define an "each" method.

And, that each method must be able to yield each item in a collection to a block. (ex: iterate through each element in an array)

Ennumerables provide many helpful functions: Searching, Sorting, Conditional, Looping & more

Examples of Enumerable Searching

array = [2,4,6,8]

>> p array.select { |x| x > 4}
=> [6,8]
// returns array with numbers greater than 4

>> p array.detect { |x| x > 4}
=> 6
// returns first detected integr greater than 4

>> p array.reject { |x| x > 4}
=> [2,4]
// returns an array without any integers greater than 4

>> p array.find_all { |x| x % 4 == 0}
=> [4,8] // returns an array with only the numbers where there are no remainders when divided by 4

Examples of Enumerable Sorting

array = [3,6,"7", "4", "1"]

>> array.sort
=> ArgumentError: comparison of Fixnum with String failed
//This will yield an error because you tried to sort integers and strings and ruby can't tell what is larger or smaller.

// But, we can use an enumberable to solve this sorting

>> array.sort_by(|x| x.to_i)
=> ["1", 3, "4", 6, "7"]
// This yields a result because we first changed all strings to integers before sorting!

Examples of Enumerable Conditionals

array = [4,8,18]

>> array.all?{|x| x.even?}
=> true
// .all? checks if every element in an array meets the conditional we set as true

>> array.all?{|x| x % 4 == 0 }
=> false
// This time .all? returns false because the 18 / 4 has a remainder of 2!

Enumerable - #reduce

array = [2,4,6]

>>array.reduce(:+)
=> 12
// .reduce will return the sume of all the numbers in the array.
// This is similar to the .inject enumberable.

>> array.inject {|sum, x| sum + x }
=> 12
// returns the sum of a memory placement, the "sum" of the first 2 elements, and adds it tot he placeholder "x", the next element, and the replaces the sum again.
// At the very end, the total sum is returned.

// Additionally you can set the initial value of the memory placement.

>> array.reduce(4, :*) => 16 or 4 (initial memory postion) + (2 + 4 + 6) (x place holder)
//Similarly...
>> array.inject(4) {|sum, x| sum + x} = > 16 or 4 (initial memory postion) + (2 + 4 + 6) (x place holder)

// This means that if we did not set an initial value for the memory position, it would be the default first element of array or range.
