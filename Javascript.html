Difference of Javascript Objects vs Ruby Hashes

To understand the similarities and differences between Javascript Objects and Ruby Hashes, we must first examine how each is creates

First, let's comapre the creation of a variable with assigned value in Javascript and Ruby.

In Javascript, we use the following code to create a varriable with an assigned value:
var apple = "core";

This code will give the variable apple an string value called "core"

In ruby, we can directly link a variable to a string;
apple = "core"

Similarly, this sets a variable apple to the string value "core".

Next, let's add values ot the objects

In Javascript, we can directly attribute our variable assigned value to an object.
//1st using the Object literal notation:
var apple = { "center": "core"};

//2nd using the new Object constructor:
var apple = new Object()
apple["center"] = "core"

In the above code, we set a key or "object" as "center" and attributed that with the value "core".

In the Object literal notation, we directly attributed the key or object :center with the string value "core".

In the new Object constructor, we linked the varaible apple with a new Object() or center: and then established the value of "center" to be "core".

Let's take a look at how to do this in ruby.
//1st using Hash Literal notation
apple = { "center" => "core"}

//2nd using Hash.new constructor
apple = Hash.new()
apple[ "center"] = "core"

As you can see, Javascripts Object literal notation and ruby's Hash literal notation have different syntaxes.

Java uses the colon ":" to associate an object (key) with a property (value).
Ruby uses a "=>" to associate a value to a key.

This occurs because the colon ":" in ruby is used to craete an immutable symbol.

So, we can have a hash with a symbol an associated value.
//In Ruby, we can make a hash with a symbol as its key.
apple = { center:"code" }

But, this syntax in ruby, creates a :center symbol with a value "code". So, it's technically different than the syntax in Javascript.

And Finally, another difference between Javascript objects and Ruby Hashes, is the addition of the "var" syntax to create a variable.

I hope this blog addresses the minor differences in syntax between Javascript Object and Ruby Hashes.
